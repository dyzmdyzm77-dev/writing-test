# UX Writing 검토 플러그인

Figma에서 선택한 영역의 텍스트를 검사해 맞춤법·띄어쓰기·톤(해요체) 수정안을 제안하고, 캔버스 위에 형광펜·코멘트로 표시해주는 플러그인입니다. 클로드(Claude) 연동으로 문구 추천·번역도 지원합니다.

## 주요 기능

### 1. 텍스트 검토 (검토하기)
- 선택한 프레임/그룹 내부의 모든 텍스트 노드를 분석합니다
- **네이버 맞춤법 검사** (비공식 API) + **로컬 규칙** (해요체 변환, 조사 을/를 교정, 띄어쓰기, 마침표, 사내 용어집)
- 변경 전/후와 변경 사유를 목록으로 보여줍니다

### 2. 캔버스 어노테이션
- 변경이 필요한 글자 구간에 노란 형광펜 표시
- 구간 위에 `원래 → 변경` 코멘트 말풍선 표시
- 프레임을 옮기면 표시도 따라가고, 항목 클릭 시 해당 위치로 이동·확대됩니다
- 적용·취소 시 자동 정리됩니다

### 3. 선택 적용 (적용하기)
- 목록에서 원하는 항목만 선택해 적용할 수 있습니다
- 글자 단위 스타일(부분 볼드 등)을 보존하면서 변경 구간만 교체합니다
- 검토 후 텍스트가 바뀌었거나 삭제된 항목은 적용하지 않고 알려줍니다

### 4. 문구 추천·번역 (클로드 다리)
- 이 PC에 로그인된 **개인 클로드 구독**으로 동작합니다 (서버 API 키 없음)
- 추천: 클로드가 스타일 가이드(ux-writing.md) 전문 + 예시 사전(recommend-examples.md)을 참고해 서로 다른 접근의 대안 3개를 제안 → 결과는 용어집 치환·네이버 맞춤법 검수를 거쳐 표시됩니다
- 번역: 한국어 ↔ 영어 자동 감지
- 다리(로컬 서버)는 추천·번역 화면에 들어가면 자동으로 켜지고, 플러그인을 닫으면 스스로 꺼집니다
- 추천 카드의 좋아요(👍)는 제보 서버에 쌓이고, `npm run sync-feedback`으로 내려받아 예시 사전에 반영할 수 있습니다

## ⚠️ 알아두기

- **검토를 실행하면 선택 영역의 텍스트가 외부로 전송됩니다.** 맞춤법 검사는 Vercel 심부름 서버와 네이버 서버를, 추천·번역은 본인 클로드 계정을 거칩니다. **외부 공개가 안 된 기밀 시안에는 사용에 주의하세요.**
- 네이버 맞춤법 검사는 **비공식 API**(py-hanspell 방식)라서 네이버가 구조를 바꾸면 동작하지 않을 수 있습니다. 이 경우 로컬 규칙(톤·조사·용어집 검사)만으로 계속 동작합니다.
- **500자가 넘는 텍스트는 맞춤법 검사를 건너뛰고** 로컬 규칙만 적용합니다.
- 로컬 규칙은 정규식 기반이라 드물게 잘못된 제안(고유명사 띄어쓰기 등)이 있을 수 있습니다. 적용 전에 목록에서 확인하세요.

## 설치 방법

### 1단계: Node.js 설치

1. [Node.js 공식 웹사이트](https://nodejs.org/ko/download/)에서 LTS 버전을 설치합니다
2. 터미널(맥) 또는 명령 프롬프트(윈도우)에서 확인합니다:
   ```
   node --version
   npm --version
   ```

### 2단계: 의존성 설치 및 빌드

이 폴더(`manifest.json`이 있는 폴더)에서 실행합니다:

```
npm install
npm run build
```

빌드가 성공하면 `code.js`가 갱신되고, 클로드 다리 감시자가 로그인 자동시작으로 등록됩니다 (Windows·macOS 모두 지원, 관리자 권한 불필요).

### 3단계: Figma에서 플러그인 실행

1. Figma 데스크톱 앱에서 디자인 파일을 엽니다
2. **Plugins** > **Development** > **Import plugin from manifest...** 선택
3. 이 폴더의 `manifest.json`을 선택합니다

### 추천·번역을 쓰려면 (선택)

- [Claude Code](https://claude.com/claude-code)(`claude` CLI)가 설치·로그인돼 있어야 합니다
- 플러그인 첫 화면에서 이 PC의 클로드 계정을 확인하고 시작합니다 — 로그인이 필요하면 버튼 한 번으로 브라우저 로그인 창이 열립니다
- **처음 쓰는 PC의 최소 설정** (한 번만): 플러그인이 처음 PC를 감지하면 로그인 화면에 설치 버튼이 바로 떠요.
  - **맥**: [클로드 설치 파일 받기] → `클로드다리-설치.zip` → 압축 풀고 나온 `클로드다리-설치.command`를 **우클릭 → [열기]** (첫 실행만 우클릭 — 인터넷에서 받은 파일이라 더블클릭은 macOS가 막습니다). 폴더가 있으면 폴더 안의 같은 파일을 실행해도 됩니다.
  - **윈도우**: [클로드 설치 파일 받기] → `클로드다리-설치.bat` 한 번 실행. 폴더가 있으면 `클로드다리-원클릭연결.vbs`도 같은 효과입니다.
  - 어느 쪽이든 **Node.js와 Claude Code(로그인)**가 있어야 하고, 없으면 설치가 단계별로 안내합니다.
  - 피그마 플러그인은 샌드박스라 설치·실행을 직접 못 합니다 — 그래서 이 한 번이 필요하고, 이후에는 로그인할 때마다 감시자가 자동으로 켜지고 추천을 누르면 클로드가 알아서 연결됩니다.

## 사용 방법

1. 검사하고 싶은 프레임/그룹/텍스트를 선택합니다
2. **검토하기**를 누르면 분석 후 수정 제안 목록과 캔버스 표시가 생성됩니다
3. 목록에서 항목을 클릭하면 해당 텍스트로 이동합니다 (체크박스로 적용 대상 선택)
4. **적용하기**를 누르면 선택한 항목이 일괄 수정되고 해당 표시가 사라집니다
5. **취소**를 누르면 캔버스 표시가 모두 제거됩니다

수정한 텍스트를 Figma에서 Ctrl+Z(⌘Z)로 되돌리면 해당 항목의 표시도 자동으로 정리됩니다.

## 말투·용어 커스터마이징 (md 파일 3종)

코드 지식 없이 md 파일만 고치면 됩니다:

| 파일 | 내용 | 반영 방법 |
|---|---|---|
| [glossary.md](glossary.md) | 용어 통일 표, 합성어 보호, 동작 명사, 예외 표기 | `npm run build` + 플러그인 재실행 |
| [ux-writing.md](ux-writing.md) | 스타일 가이드 전문 (해요체·능동·긍정 + 예외 규칙) — 클로드 추천 프롬프트에 포함 | 다리 재시작 ([클로드] 버튼 껐다 켜기) |
| [recommend-examples.md](recommend-examples.md) | 추천 예시 사전 (클로드 few-shot 교재) | `npm run build` + 다리 재시작 |

- "X → X 같이 똑같아 보이는 제안"이 나오면 → 그 단어를 glossary.md **합성어 보호**에 추가
- 예외 처리가 필요한 복잡한 규칙(특정 단어 제외 등)은 `code.ts`의 `TERM_RULES`에 있습니다

## 개발 모드 (코드 수정 시)

```
npm run watch   # 저장할 때마다 자동 컴파일
npm run lint    # ESLint 검사
```

코드를 수정한 뒤에는 Figma에서 플러그인을 다시 실행하면 반영됩니다. 다리(`scripts/claude-bridge.js`)를 수정했다면 다리도 재시작해야 합니다 — 동작이 바뀌는 수정이면 `BRIDGE_V`와 code.ts의 `BRIDGE_MIN_V`를 함께 올리세요 (플러그인이 구버전 다리를 감지해 [업데이트 필요] 버튼을 띄웁니다).

## 문제 해결

### "네이버 맞춤법 미작동" 토스트가 뜨는 경우
- 네이버가 비공식 API 구조를 바꿨을 수 있습니다 (이 경우 로컬 규칙만 동작)
- 사내망이라면 Vercel(`report-admin-amber.vercel.app`) 접근이 차단되지 않았는지 확인하세요

### 추천·번역이 안 되는 경우
- [클로드] 버튼이 ⚫ 꺼짐이면 눌러서 켜세요 (감시자가 죽어 있으면 `npm run build` 한 번으로 재등록)
- 🟠 로그인 필요면 버튼을 눌러 브라우저에서 로그인하세요
- 🟠 업데이트 필요면 버튼을 눌러 다리를 새 코드로 재시작하세요

### 플러그인이 실행되지 않는 경우
- `npm run build`로 `code.js`가 생성됐는지 확인하세요
- Figma를 재시작하거나 플러그인을 다시 import 해보세요

### 텍스트가 분석되지 않는 경우
- 텍스트가 들어 있는 프레임/그룹을 선택했는지 확인하세요
- 잠겨 있거나 숨겨진(visible off) 레이어는 검사에서 제외됩니다

## 기술 스택

- **TypeScript** + **@figma/plugin-typings** (Figma Plugin API)
- **HTML/CSS/JavaScript** (플러그인 UI)
- **Vercel** (네이버 passportKey 중계 + 제보 저장, `ux-writing-reports` 저장소)
- **Claude Code CLI** (문구 추천·번역 — 로컬 다리 `scripts/claude-bridge.js`가 세션 유지)
