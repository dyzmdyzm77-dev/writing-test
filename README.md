# UX Writing 톤 분석 플러그인

Figma 내 텍스트 노드를 UX Writing 톤 프로파일과 비교하여 불일치하는 텍스트를 자동으로 감지하고 선택하는 플러그인입니다.

## 주요 기능

### 1. UX Tone Profile 설정
- **톤 (Tone)**: 친근한, 캐주얼, 격식체, 전문적 중 선택
- **문장 유형 (Sentence Type)**: 명령형, 서술형, 의문형, 제한 없음
- **이모지 사용 여부**: 이모지 사용/미사용 설정
- **존댓말 사용 여부**: 존댓말 사용/미사용 설정
- **길이 제한**: 최대/최소 길이 설정 (선택사항)

### 2. 텍스트 분석 및 불일치 감지
- 선택한 영역 또는 전체 페이지의 텍스트 노드를 자동 분석
- UX Writing 가이드와 일치하지 않는 텍스트를 자동으로 감지
- 불일치 사유를 상세히 표시 (톤, 문장 유형, 이모지, 존댓말, 길이 등)
- 유사도 점수 제공 (0-100%)

### 3. 자동 선택 및 하이라이팅
- 불일치하는 텍스트 노드를 자동으로 선택
- Figma 캔버스에서 해당 노드로 자동 이동 및 확대

### 4. 수정 제안 및 자동 적용
- 불일치하는 텍스트에 대한 수정 제안 제공
- "적용하기" 버튼으로 한 번에 수정 가능

## 설치 방법 (비개발자용)

### 1단계: Node.js 설치

1. [Node.js 공식 웹사이트](https://nodejs.org/ko/download/)에 접속합니다
2. 운영체제에 맞는 버전을 다운로드합니다 (LTS 버전 권장)
3. 다운로드한 파일을 실행하여 설치합니다
4. 설치가 완료되면 터미널(맥) 또는 명령 프롬프트(윈도우)를 열고 다음 명령어로 확인합니다:
   ```
   node --version
   npm --version
   ```
   버전 번호가 표시되면 설치가 완료된 것입니다.

### 2단계: TypeScript 설치

터미널(맥) 또는 명령 프롬프트(윈도우)를 열고 다음 명령어를 입력합니다:

```
npm install -g typescript
```

### 3단계: 프로젝트 의존성 설치

1. 터미널 또는 명령 프롬프트를 엽니다
2. 이 프로젝트 폴더로 이동합니다:
   ```
   cd "/Users/jihoonkim/Desktop/UX Writing Test"
   ```
   (윈도우의 경우 경로가 다를 수 있습니다)
3. 다음 명령어를 실행합니다:
   ```
   npm install
   ```

### 4단계: TypeScript 컴파일

프로젝트 폴더에서 다음 명령어를 실행합니다:

```
npm run build
```

컴파일이 성공하면 `code.js` 파일이 생성됩니다.

### 5단계: Figma에서 플러그인 실행

1. Figma 데스크톱 앱을 엽니다
2. 디자인 파일을 엽니다
3. 상단 메뉴에서 **Plugins** > **Development** > **Import plugin from manifest...**를 선택합니다
4. 이 프로젝트 폴더의 `manifest.json` 파일을 선택합니다
5. 플러그인이 목록에 추가됩니다

## 사용 방법

### 기본 사용 흐름

1. **톤 프로파일 설정**
   - 플러그인을 실행합니다
   - 원하는 UX Writing 톤 프로파일을 설정합니다
   - "프로파일 저장" 버튼을 클릭하여 저장합니다

2. **텍스트 선택**
   - 분석하고 싶은 텍스트 레이어를 선택합니다
   - 또는 "전체 페이지" 옵션을 선택하여 페이지 전체를 분석합니다

3. **분석 실행**
   - "분석 실행" 버튼을 클릭합니다
   - 플러그인이 텍스트를 분석하고 불일치하는 노드를 자동으로 선택합니다

4. **결과 확인 및 수정**
   - 분석 결과에서 불일치 사유를 확인합니다
   - 수정 제안이 있는 경우 "적용하기" 버튼을 클릭하여 자동 수정합니다

### 분석 범위 선택

- **선택 영역**: 현재 선택한 노드 내의 텍스트만 분석
- **전체 페이지**: 현재 페이지의 모든 텍스트 노드를 분석

### 불일치 사유 종류

플러그인은 다음 항목들을 검사합니다:

1. **톤 불일치**: 설정한 톤(친근한/캐주얼/격식체/전문적)과 일치하지 않음
2. **문장 유형 불일치**: 설정한 문장 유형(명령형/서술형/의문형)과 일치하지 않음
3. **이모지 사용**: 설정과 다르게 이모지가 포함되어 있거나 없음
4. **존댓말 사용**: 설정과 다르게 존댓말이 포함되어 있거나 없음
5. **길이 제한**: 설정한 최대/최소 길이를 초과하거나 미달

### 유사도 점수

- **70% 이상**: 높은 일치도 (녹색)
- **40-69%**: 중간 일치도 (주황색)
- **40% 미만**: 낮은 일치도 (빨간색)

## 변환 규칙 예시

플러그인이 자동으로 변환하는 패턴 예시:

### 친근한 톤으로 변환
- `확인해보세요` → `확인하기`
- `저장해주세요` → `저장하기`
- `삭제하시겠어요` → `삭제하기`
- `업로드하시기 바랍니다` → `업로드하기`

### 감지되는 패턴
- **격식체**: `합니다`, `하십시오`, `하시기 바랍니다`, `입니까`
- **비격식체**: `해`, `해요`, `할래`, `할게`
- **친근한 톤**: `하기`, `해보기`, `해주기`
- **존댓말**: `하세요`, `해주세요`, `하시겠어요`, `하시겠습니까`

## 개발 모드에서 사용하기 (코드 수정 시)

코드를 수정하고 테스트하려면:

1. 터미널에서 다음 명령어를 실행합니다:
   ```
   npm run watch
   ```
   이 명령어는 코드를 자동으로 컴파일합니다.

2. Visual Studio Code를 사용하는 경우:
   - **Terminal** > **Run Build Task...** 메뉴를 선택합니다
   - **npm: watch**를 선택합니다
   - 이제 코드를 수정하면 자동으로 컴파일됩니다

## 문제 해결

### 컴파일 오류가 발생하는 경우

- Node.js와 TypeScript가 제대로 설치되었는지 확인하세요
- `npm install` 명령어를 다시 실행해보세요
- 터미널에서 `npm run build`를 실행하여 오류 메시지를 확인하세요

### 플러그인이 실행되지 않는 경우

- `code.js` 파일이 생성되었는지 확인하세요
- Figma를 재시작해보세요
- 플러그인을 다시 import 해보세요

### 텍스트가 분석되지 않는 경우

- 텍스트 레이어를 선택했는지 확인하세요
- 텍스트가 한국어인지 확인하세요
- 폰트가 로드되지 않은 경우, 텍스트를 한 번 편집해보세요

### 불일치 노드가 선택되지 않는 경우

- 분석 결과에서 불일치 노드가 있는지 확인하세요
- Figma의 선택 기능이 정상 작동하는지 확인하세요
