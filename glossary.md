# UX Writing 용어집

플러그인이 검사에 사용하는 용어 목록입니다. **이 파일을 수정한 뒤 터미널에서 `npm run build`를 실행하고, Figma에서 플러그인을 다시 실행하면 반영됩니다.**

- 표/목록만 수정하면 됩니다. 제목(`##`)은 바꾸지 마세요.
- 예외 처리가 필요한 복잡한 규칙(예: "암호화는 제외", "얼굴(지문) + 카드" 패턴)은 `code.ts`의 `TERM_RULES`에 있습니다.

## 용어 통일

왼쪽 "기존" 표현을 발견하면 오른쪽 "권장" 표현으로 바꾸라고 제안합니다. 위에서부터 순서대로 적용됩니다.

| 기존 | 권장 |
|------|------|
| 개인 사용자 조회 | 개별 사용자 조회 |
| 자격선택 | 권한설정 |

| 자격별 | 권한별 |
| 관리자 리스트 | 관리자 목록 |
| 총 사용자 | 전체 사용자 |
| 재부팅 | 재시작 |
| 에러 | 오류 |

## 권장 문구

말투·어미를 다듬는 규칙입니다 (ux-writing.md 가이드 기반). 검토 결과에 "권장 문구" 칩으로 표시됩니다. 위 "용어 통일"과 형식은 같지만, 도메인 용어가 아니라 톤 규칙을 여기에 둡니다.

| 기존 | 권장 |
|------|------|
| 되어요 | 돼요 |
| 되었어요 | 됐어요 |
| 되었습니다 | 됐어요 |
| 하시겠어요 | 할까요 |
| 계시나요 | 있나요 |
| 여쭤볼게요 | 확인할게요 |
| 보냅니다 | 보내요 |

## 예외 표기

네이버 맞춤법이 표준 표기로 바꾸려고 해도 우리 표기를 그대로 유지할 단어들입니다. 원문에 "유지할 표기"가 쓰여 있으면 네이버의 교정을 되돌립니다. (원문에 표준 표기가 쓰여 있으면 그대로 둡니다 — 양쪽 다 허용)

| 유지할 표기 | 네이버가 바꾸는 표기 |
|------|------|
| 렌탈 | 렌털 |
