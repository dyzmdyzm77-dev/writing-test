# S-1 S/W UX Writing Assistant (Figma 플러그인)

에스원 UX Writing 검토 플러그인. 선택 영역의 텍스트를 네이버 맞춤법 + 로컬 규칙(용어집/해요체/띄어쓰기)으로 검사하고, 캔버스에 형광펜·코멘트를 표시한다.

## 빌드/실행

- `npm install` (최초 1회) → `npm run build` → Figma에서 Plugins > Development > Import plugin from manifest
- `npm run build` = glossary.md를 code.ts에 주입(`scripts/build-glossary.js`) + tsc 컴파일. **code.ts의 GLOSSARY:BEGIN/END 마커 영역은 자동 생성 — 직접 수정 금지**
- ui.html은 빌드 불필요 (플러그인 재실행만)
- 검증은 vm 샌드박스로: code.js를 `vm.runInContext`에 figma 스텁과 함께 로드해 `suggestFriendlyKorean(text, naverChecked)` 호출

## 용어집 워크플로우

- 단순 치환/합성어 보호/동작 명사/예외 표기는 **glossary.md에서 편집** → `npm run build`
- 가드(lookahead)가 필요한 규칙만 code.ts의 TERM_RULES에 직접 작성
- "X → X처럼 똑같아 보이는 제안"이 나오면 → 네이버가 합성어를 띄어 쓴 것 → glossary.md "합성어 보호"에 단어 추가
- 네이버가 표기를 바꾸는 단어(렌탈→렌털 등) → "예외 표기" 표에 추가

## 피드백 환류 워크플로우

- 추천 카드의 좋아요(👍)는 제보 저장소(report-admin)에 `reason='추천 좋아요'` 마커로 저장됨 — 이 문자열은 code.ts(LIKE_SUGGESTION)와 scripts/sync-feedback.js가 공유하므로 한쪽만 바꾸면 안 됨
- `npm run sync-feedback` → 좋아요+오수정 제보를 내려받아 `feedback-candidates.md` 생성(커밋 금지, gitignore됨) → 사람이 골라 recommend-examples.md/ux-writing.md에 옮기고 `npm run build`
- 사내 프록시 때문에 node fetch에 `--use-env-proxy` 필요 (npm 스크립트에 포함됨, Node 24+)

## 핵심 설계 결정 (바꾸기 전에 읽을 것)

- **띄어쓰기는 네이버 우선**: 부사 띄어쓰기 정규식(ADVERB_SPACING_RULES)은 네이버 실패 시 폴백 전용. COMPOUND_PROTECT_RULES가 네이버의 합성어 분리를 되돌림
- **네이버 검사는 배치**(여러 문구를 \n으로 묶어 요청 1개) + 세션 캐시(naverCache). 줄바꿈은 <br>로 보존됨(실서버 확인). 공백 구조는 alignWhitespace로 원문에 정렬 (U+2028 등 모든 줄바꿈 문자 처리 — \n만 가정하지 말 것)
- **위치 추적은 폴링 금지**: dynamic-page라 figma.on('documentchange')는 등록 실패 → `figma.currentPage.on('nodechange')` 사용. setInterval 폴링은 수천 개 어노테이션에서 캔버스 렉 유발 (제거된 이력 있음)
- **동기 getNodeById 금지** (dynamic-page에서 throw) → getNodeByIdAsync
- '해 주세요' 띄어쓰기: 동작 명사(glossary.md 목록)면 '확인해 주세요', 부사면 '같이 해주세요'
- 검토 사유는 키워드 칩(맞춤법/띄어쓰기/해요체/간결하게/용어 통일 등 10종)으로 표시
- **추천·번역은 클로드 다리 전용** (2026-07 Gemini/API 키 경로 전부 제거 — 키 모달·사용량 카운터·serverSharedKeyMissing 없음). 추천: 다리 `/recommend`, 다리가 꺼져 있으면 예시·규칙 폴백. 번역: 다리 `/translate`(같은 세션, 턴별 지시로 JSON 객체 {translated, direction} 요구 — 줄 수 유지 지시 포함), 다리 없으면 토스트 안내만. 추천·번역 화면 진입 클릭(홈카드·상단탭)이 **제스처 안에서** 다리를 자동으로 켠다(ensureBridgeFromGesture — 비동기 응답 뒤 window.open은 팝업 차단됨). 예시 사전은 화면에 안 나오고 AI few-shot 교재로만 쓰임. 예시 사전의 추천안은 검토 규칙과 모순되면 안 됨(마침표·'해 주세요' 띄어쓰기 포함) — vm 샌드박스로 suggestFriendlyKorean(추천안)이 무변경인지 확인
- **클로드 다리(scripts/claude-bridge.js)**: `claude -p` stream-json 세션을 상시 유지하는 로컬 서버(localhost:11888). 켜기: 클로드다리-켜기.bat, 끄기: 클로드다리-끄기.bat(POST /shutdown). (클로드다리-자동시작등록.vbs는 제거됨 — 다리 자체를 로그인마다 자동 기동하는 구버전 방식으로, 감시자 온디맨드 설계·비용 원칙과 충돌) **깨우기 주경로는 감시자(scripts/bridge-watcher.js, 포트 11889)**: 피그마 데스크톱이 플러그인의 claudebridge:// 열기를 window.open/숨은 iframe/figma.openExternal **셋 다 소리 없이 막는 것이 실측 확인됨**(2026-07, 4분 프로세스 감시로 wscript 도달 0회) — 유일하게 안 막히는 통로는 fetch. 그래서 초소형 상주 감시자(~15MB, claude 안 물어 클로드 앱 업데이트 안 막음)가 POST /wake를 받으면 다리를 켠다. 로그인 자동시작(HKCU Run, ClaudeBridgeWatcher)으로 등록, 사용자 승인 받음. WAKE_BRIDGE는 **순차 폴백**(2026-07 병행→순차 전환): 감시자 fetch 성공 시 거기서 끝, 실패했을 때만 openExternal(claudebridge://) 보조 — 병행하면 프로토콜이 안 막힌 피그마에서 다리가 이중 기동돼 창이 보일 수 있었음. 같은 이유로 ui.html의 window.open/숨은 iframe 프로토콜 열기도 제거(openBridgeProtocol은 WAKE_BRIDGE 위임만). **창 안 뜨게 하는 규칙(win32)**: 감시자의 다리 스폰은 node 직접 + windowsHide(CREATE_NO_WINDOW), detached 금지 — cmd/vbs 경유·detached+windowsHide 조합은 콘솔 창이 사용자에게 노출됨(실측). 숨은 콘솔을 claude 자식이 물려받아 어떤 창도 안 생긴다. 감시자 /health의 v 필드로 구버전 프로세스 잔존 확인(v2=창숨김판). manifest devAllowedDomains에 11889 추가됨(manifest 변경은 플러그인 재실행 필요). **첫 PC 설정은 npm이 자동으로 해결**: `npm install`/`npm run build`가 scripts/register-protocol.js를 실행해 claudebridge:// 등록 + 감시자 자동시작 등록 + 감시자 즉시 기동(관리자 불필요, fail-soft). **macOS도 지원(2026-07)**: register-protocol.js가 darwin이면 launchd LaunchAgent(com.claudebridge.watcher)로 감시자만 등록한다 — claudebridge://는 등록 안 함(피그마가 프로토콜을 막아 감시자 fetch가 유일 경로라 불필요). plist에 등록 시점 PATH를 굳혀 넣는 것이 핵심(launchd 기본 PATH엔 /usr/local/bin이 없어 claude를 못 찾음). 감시자의 다리 스폰은 전 OS에서 process.execPath 직접 스폰 (win32는 windowsHide, 상세 규칙은 위 WAKE_BRIDGE 항목). killProc은 POSIX에서 detached 스폰 + 프로세스 그룹 kill(-pid)로 claude 트리를 정리(taskkill /T 대응 — 2026-07 Mac에서 고아 0 실측). 추천·번역은 개인 클로드 구독 전용이 확정 노선 — 서버 API 키·Gemini 경로 재도입 금지. npm 없이 빌드된 폴더만 복사받은 PC를 위한 폴백: 켜기 버튼이 20초 안에 실패하면 버튼이 [🔧 클로드 설치 파일 받기]로 바뀌고, 누르면 **클로드다리-설치.bat**을 다운로드로 내려준다(GET_INSTALLER → installer-file, code.ts INSTALLER:BEGIN/END 마커에 build가 base64 주입). 이 bat은 자기완결형(다리 js+예시 md+런처 vbs를 base64 payload로 내장, `exit /b` 뒤 ::PS::/::BRIDGE::/::EXAMPLES::/::LAUNCHER::/::END:: 섹션): 실행하면 %LOCALAPPDATA%\ClaudeBridge에 설치 + claudebridge:// 등록 + node/claude 점검(없으면 단계별 팝업: nodejs.org 열기 / claude 설치·로그인 안내) + 다리 즉시 켜기까지 전부 한다. CB_SILENT=1이면 팝업 생략(자동 테스트용). bat은 npm run build가 저장소 루트에 생성 — 직접 수정 금지. **프로토콜 등록만큼은 플러그인 버튼이 직접 못 함**(피그마 샌드박스 — 레지스트리/실행 능력 없음, 우회로 없음) — 그래서 "다운로드 → 한 번 실행" 흐름이 최소치. 저장소 폴더가 있는 PC는 클로드다리-원클릭연결.vbs 한 번 실행도 동일 효과(등록+점검+켜기, 단 저장소 경로 기준). claudebridge://는 claude-bridge-silent.vbs(점검 런처)를 실행. 두 vbs와 설치 PS의 node/claude 점검 로직은 세쌍둥이 — 같이 유지. vbs는 반드시 UTF-16LE로 저장([[vbs-must-be-utf16le]]). 속도 핵심 3가지 — UI 제거 — getSelectedModel()이 'opus' 상수 반환, 다리 기본값도 opus). 다리는 요청의 model이 바뀌면 세션 재시작(재워밍업 ~18초, 같은 모델은 5~10초). 중복 실행 시 EADDRINUSE 감지해 새 인스턴스가 조용히 종료. **구버전 다리 감지(2026-07)**: 코드를 pull·복사해도 **이미 떠 있는 다리는 옛 코드 그대로**라 껐다 켜기 전엔 새 동작이 안 나온다(맥만 고쳐진 것처럼 보이는 착시의 원인 — 실제 발생). 그래서 /health가 v(BRIDGE_V)·dir(__dirname)을 노출하고, code.ts의 BRIDGE_MIN_V보다 낮거나 v가 없으면 problem='bridge-old' → 버튼 [🟠 업데이트 필요] → 클릭 시
